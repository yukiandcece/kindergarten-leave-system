# 幼儿园离园考勤管理系统 - 产品文档

## 📋 产品概述

幼儿园离园考勤管理系统是一套专门为幼儿园设计的离园安全管理解决方案，通过手机号验证登录和NFC刷卡技术，实现幼儿离园全流程的数字化管理和安全确认。

## 🎯 核心功能

### 1. 登录验证系统

#### 1.1 手机号登录
- **输入限制**：11位手机号码
- **实时验证**：自动过滤非数字字符
- **格式要求**：1开头的11位有效手机号

#### 1.2 验证码登录
- **验证码格式**：4位数字验证码
- **获取验证码**：
  - 验证手机号格式正确性
  - 60秒倒计时防重复发送
  - 倒计时期间显示剩余秒数
  - 倒计时结束自动恢复按钮状态
  - 当前为演示模式，未连接真实短信服务
- **输入限制**：仅允许输入数字，最多4位

#### 1.3 终端选择
系统支持三种工作终端：
- **入园考勤拍摄端**：演示模式
- **入园考勤确认端**：演示模式
- **离园考勤确认端**：跳转至离园考勤详情页面

**跳转逻辑**：
- 选择"离园考勤确认端"并登录成功后，自动跳转至离园详情页面
- 其他终端登录成功后显示演示提示

#### 1.4 状态显示
- 实时显示当前选择的终端名称
- 终端切换时自动更新显示文本
- 终端选择信息会保存到本地存储

#### 1.5 登录验证
- **手机号验证**：必须为11位有效手机号
- **验证码验证**：必须为4位数字验证码
- **登录反馈**：验证失败时弹出错误提示
- **演示模式**：当前为演示版本，未连接真实后端服务

### 2. 离园考勤管理

#### 2.1 NFC刷卡识别
- **NFC卡号格式**：NFC + 16位数字（如：NFC6230559802392803）
- **刷卡功能**：
  - 模拟NFC刷卡（点击按钮生成随机NFC卡号）
  - 自动显示NFC卡号
  - 显示学生姓名和NFC卡号确认信息
- **提示信息**：`NFC卡号 [卡号]，学生 [姓名]，请老师确认接送人`

#### 2.2 接送人信息展示
**显示内容**：
- 学生姓名
- 接送人姓名
- 与学生关系（爸爸、妈妈、爷爷、奶奶）
- 接送人人脸照片

**展示数量**：每位学生最多显示4个授权接送人

**人脸照片功能**：
- 点击照片可放大查看
- 显示接送人姓名作为图片标题
- 支持ESC键关闭大图

#### 2.3 接送确认流程
1. **刷卡识别**：NFC刷卡后显示授权接送人列表
2. **查看信息**：老师查看接送人详细信息和人脸照片
3. **确认接送人**：点击对应接送人的"确认接送人"按钮
4. **完成确认**：
   - 自动记录离园信息（时间、幼儿、接送人、关系、结果）
   - 显示确认成功提示：`已确认 [幼儿姓名] 由 [接送人姓名]（[关系]）接送离园`
   - 清空NFC卡号输入框
   - 清空接送人列表
   - 重置学生信息
   - 禁用异常处理按钮和添加记录按钮
   - 准备下一次操作

#### 2.4 离园记录管理
**记录信息包含**：
- 离园时间（精确到秒）
- 幼儿姓名
- 接送人姓名
- 与幼儿关系
- 离园结果（已确认离园）

**分页显示**：
- 每页显示5条记录
- 显示当前页码和总页数
- 支持上一页/下一页翻页
- 自动禁用无效翻页按钮

### 3. 异常处理系统

#### 3.1 无授权关系处理
**触发条件**：刷卡成功后可启用
**功能描述**：
- 点击"无授权关系/人脸不匹配，添加备注"按钮
- 弹出系统输入框要求填写备注信息
- 记录异常情况到异常列表
- 格式：`时间 幼儿姓名 无授权关系：备注内容`
- 处理完成后自动禁用异常处理按钮

#### 3.2 紧急联系人通知
**触发条件**：刷卡成功后可启用
**功能描述**：
- 点击"通知紧急联系人"按钮
- 自动记录通知操作
- 格式：`时间 幼儿姓名 已通知紧急联系人`

#### 3.3 异常记录管理
- **时间记录**：精确到秒的时间戳
- **幼儿信息**：关联当前刷卡的学生姓名
- **异常描述**：详细记录异常类型和处理情况
- **列表显示**：按时间倒序排列

### 4. 手动添加离园记录

#### 4.1 功能触发
- **启用条件**：NFC刷卡成功后可用
- **按钮位置**：离园记录板块右上角

#### 4.2 记录表单
**弹窗功能**：
- 点击弹窗外部区域可关闭
- 表单重置功能
- 输入框自动聚焦

**必填字段**：
- 幼儿姓名（自动填充当前学生，只读）
- 接送人姓名（手动输入，必填）
- 关系选择（下拉选择：爸爸、妈妈、爷爷、奶奶、外公、外婆、其他，必选）

#### 4.3 提交处理
- **表单验证**：接送人姓名和关系为必填项，未填写时弹出错误提示
- **记录添加**：自动生成带时间戳的完整记录
- **成功提示**：显示成功信息并更新状态提示文本
- **界面重置**：关闭弹窗并重置表单，准备下一次操作
- **状态更新**：提示文本显示格式：`已添加离园记录：[幼儿姓名] 由 [接送人]（[关系]）接送离园`

### 5. 图片查看功能

#### 5.1 人脸照片大图查看
- **触发方式**：点击接送人卡片中的人脸照片
- **显示内容**：
  - 人脸照片（大图）
  - 接送人姓名（作为图片标题）
- **关闭方式**：
  - 点击关闭按钮（×）
  - 点击模态框背景区域
  - 按ESC键

## 🔄 业务流程

### 标准离园流程
```
1. 家长到场 → 持NFC卡/设备
2. NFC刷卡 → 系统识别卡号
3. 信息展示 → 显示学生信息和授权接送人列表
4. 身份确认 → 老师查看人脸照片确认接送人身份
5. 点击确认 → 选择正确的接送人并确认
6. 记录完成 → 系统自动记录离园信息
7. 界面重置 → 准备迎接下一位学生
```

### 异常处理流程
```
1. 情况识别 → 人脸不匹配或无授权关系
2. 启用异常处理 → 刷卡后异常处理按钮变为可用状态
3. 记录异常 → 选择对应的异常处理方式
4. 填写备注 → 详细说明异常情况
5. 通知处理 → 通知紧急联系人或其他处理流程
6. 记录存档 → 异常情况自动记录到系统
```

### 手动添加流程
```
1. NFC刷卡 → 获取学生基本信息
2. 点击添加 → 点击"添加离园记录"按钮
3. 填写信息 → 输入接送人姓名和关系
4. 提交表单 → 系统验证并保存记录
5. 完成操作 → 显示成功提示并关闭弹窗
```

## 🔘 按钮状态管理

### 状态控制逻辑
系统根据操作流程自动管理相关按钮的启用/禁用状态：

#### 启用状态（刷卡成功后）
- **无授权关系/人脸不匹配，添加备注**按钮
- **通知紧急联系人**按钮
- **添加离园记录**按钮

#### 禁用状态（初始状态和确认完成后）
- 所有异常处理按钮
- 添加记录按钮
- 防止重复操作

### 状态同步
- 刷卡成功：同时启用上述三个按钮
- 确认接送人：同时禁用上述三个按钮
- 异常处理完成：禁用所有异常处理按钮
- 添加记录完成：禁用添加记录按钮

## ⚠️ 系统限制

### 当前版本限制
1. **数据持久化**：所有数据仅保存在页面会话中，刷新页面后清空，不支持数据保存到服务器
2. **用户验证**：手机号验证码登录为演示模式，未连接真实短信服务
3. **NFC硬件**：当前为模拟NFC刷卡功能，未集成真实NFC读卡器硬件
4. **数据源**：学生和接送人信息为系统随机生成的演示数据
5. **数据导出**：不支持离园记录导出功能（Excel、PDF等）
6. **多用户**：不支持多老师同时操作，数据不共享
7. **权限管理**：暂未实现不同角色权限控制

### 技术限制
1. **浏览器兼容性**：需要现代浏览器支持（Chrome、Firefox、Safari、Edge）
2. **网络依赖**：页面资源加载需要网络连接
3. **数据同步**：不同设备/浏览器间数据无法同步
4. **历史记录**：页面刷新后历史记录清空，无法恢复

### 使用限制
1. **并发处理**：同一时间只能处理一个学生的离园流程
2. **数据容量**：当前会话中的记录数量受浏览器内存限制
3. **备份恢复**：不支持数据备份和恢复功能

## 🔧 技术特性

### 输入验证
- **手机号**：11位数字，1开头，实时过滤非数字字符
- **验证码**：4位数字，实时过滤非数字字符，限制长度
- **NFC卡号**：NFC + 16位数字，系统自动生成
- **接送人姓名**：任意文本输入，必填验证
- **关系选择**：下拉菜单选择，必选验证

### 数据格式
- **时间格式**：YYYY-MM-DD HH:mm:ss（精确到秒）
- **NFC卡号格式**：NFC前缀 + 16位随机数字
- **关系类型**：爸爸、妈妈、爷爷、奶奶、外公、外婆、其他
- **学生姓名**：预定义的5个演示学生姓名

### 交互特性
- **实时验证**：输入时自动过滤非法字符和长度控制
- **防重复提交**：60秒倒计时机制防止重复获取验证码
- **状态管理**：按钮根据操作流程自动启用/禁用
- **错误提示**：输入验证失败时使用原生alert弹窗提示
- **输入聚焦**：弹窗打开时自动聚焦到输入框
- **键盘支持**：ESC键关闭模态框，Enter键提交表单

### 前端技术
- **HTML5**：语义化标签，表单验证
- **CSS3**：响应式设计，动画效果，现代布局
- **原生JavaScript**：无框架依赖，模块化设计
- **SVG**：动态生成人脸图片，无需外部图片资源

## 📱 使用指南

### 快速上手
1. **打开系统**：访问系统地址，进入登录页面
2. **选择终端**：点击"离园考勤确认端"标签
3. **输入信息**：
   - 输入11位手机号码（任意1开头的11位数字）
   - 点击"获取验证码"按钮（60秒倒计时）
   - 输入4位验证码（任意4位数字）
4. **登录系统**：点击"立即登录"按钮
5. **进入页面**：自动跳转至离园考勤详情页面

### 日常操作流程
1. **等待家长**：家长持NFC设备/卡片到场
2. **NFC刷卡**：
   - 点击"模拟NFC刷卡"按钮
   - 系统生成随机NFC卡号（如：NFC6230559802392803）
   - 显示学生姓名确认信息
3. **查看信息**：
   - 查看接送人列表（最多4个授权接送人）
   - 查看接送人详细信息和关系
   - 点击人脸照片可放大查看
4. **确认身份**：
   - 核对接送人人脸照片
   - 确认接送人关系是否正确
5. **完成确认**：
   - 点击对应接送人的"确认接送人"按钮
   - 查看确认成功提示信息
   - 系统自动清空，准备下一次操作

### 异常处理流程
1. **识别异常**：
   - 人脸照片不匹配
   - 接送人无授权关系
   - 其他需要备注的情况
2. **选择处理方式**：
   - 点击"无授权关系/人脸不匹配，添加备注"按钮
   - 或点击"通知紧急联系人"按钮
3. **记录异常**：
   - 在弹出的输入框中填写详细备注信息
   - 点击确定提交异常记录
4. **查看记录**：
   - 异常情况自动记录到页面下方的异常列表
   - 包含时间、学生姓名、异常描述信息

### 手动添加记录流程
1. **NFC刷卡**：先进行NFC刷卡获取学生信息
2. **打开添加界面**：
   - 点击"离园记录"板块右上角的"添加离园记录"按钮
   - 弹出添加记录弹窗
3. **填写信息**：
   - 学生姓名自动填充（只读）
   - 输入接送人姓名（必填）
   - 选择与学生的关系（必选）
4. **提交记录**：
   - 点击"确认添加"按钮
   - 系统自动生成带时间戳的离园记录
   - 查看成功提示信息

### 离园记录查询
1. **查看记录**：
   - 页面下方显示离园记录表格
   - 每页显示5条记录
   - 包含时间、幼儿、接送人、关系、结果信息
2. **翻页浏览**：
   - 点击"上一页"或"下一页"按钮
   - 查看页码信息："第 X / Y 页"
   - 自动禁用无效翻页按钮

本文档描述的系统为**演示版本**，具有以下特点：
- **功能完整**：所有交互功能均为真实可用的功能
- **模拟数据**：学生信息、接送人信息、NFC卡号均为系统随机生成
- **本地运行**：所有数据处理在浏览器端完成，无需服务器支持
- **会话限制**：数据仅在当前页面会话中有效，刷新页面后清空
- **演示用途**：主要用于功能展示、用户培训、需求验证

### 实际部署建议
如需在实际环境中使用，建议：
1. **后端集成**：连接真实数据库和用户管理系统
2. **NFC硬件**：集成真实NFC读卡器设备
3. **短信服务**：接入真实短信验证码服务
4. **数据持久化**：实现服务器端数据存储
5. **用户权限**：添加完整的用户权限管理系统
6. **数据备份**：实现数据备份和恢复功能
